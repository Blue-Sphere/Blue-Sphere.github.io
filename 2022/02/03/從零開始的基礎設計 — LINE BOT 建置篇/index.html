從零開始的基礎設計 — LINE BOT 建置篇

發表於2022-02-03|更新於2022-07-15
|閱讀量:

前言

做為一位菜鳥coder，我相信應該不少人學習的第一個程式語言都是從Python開始，Python對於初學者來說算是非常友善，語法簡單明瞭又好上手，但雖然都稱為「程式」，初學者多少都會認為總是在終端機（Terminal）運行相對來說少了這麼一點點樂趣，然而第一個有趣的想法就這樣誕生了，就是讓Python搭配Linebot創造出多種可能，所以今天來與coder們分享我所學的一些歷程，希望大家（以後的自己）可以因為這篇文章在未來設計LineBot的時候少踩一些坑。

本次使用工具

雖然文章標題是「從零開始的基礎設計 — LINE BOT 建置篇」，但基本的Visual Studio code和Python等安裝項目這裡目前就不再多贅述，網路上有許多這類的教學，如果有需要的話我也會再另外撰寫文章來說明這些軟體的安裝與細項。

  • Visual Studio Code（AKA.Vscode）
  • Python
  • Deta Cloud

正式開始

首先第一步驟就是要創辦一個Line的機器人，因為LineBot隸屬於Line的服務之一，所以開始之前必須有Line的帳號才行，有了Line的帳號後，就可以進到Line Developers（Line開發者服務）裡創辦LineBot。

建立新LineBot

進入Line Developers（Line 開發者服務）後，點擊下方的creat就能創建提供者，這裡我將LineBot的提供者命名為BlueSphere做為範例，當然一個Line帳號也能有很多個提供者。

按下creat後頁面上會出現四種機器人類型供使用者選擇，因為我們必須將LineBot串聯API與使用者互動，所以這裡要選擇第二種，也就是creat a Message API channel，老實說除了第二種其他的類型我都還沒探索過，希望將來有機會透過其他實作認識它們。

接著就來到了LineBot資訊設定的部分，這邊的Channel name指的就是LineBot的名稱，填好自己想填的名稱後就可以填寫下方的說明、選擇類別、子類別等選項，之後同意下方的Line的使用服務條款後按下creat，一個全新的LineBot就建立好了！


建立好之後我們需要手動設定一下LineBot的設置，從主頁面的Messaging API選項中，找到Auto-reply messages，點選Edit後會進到後會進到Line Offical Account Manager，在進階設定中將自動回應訊息停用，並將Webhock啟用，將自動回應訊息停用是因為需要透過API串聯訊息的回覆，所以必須停用，而啟用Webhook的原因稍後才會需要做到設置，這邊先不再多贅述。

官方釋出的LineBot API建置

完成LineBot的設置後，下一步就是要去官方釋出的LINE Messaging API SDK for Python逛一逛了，官方的用法是採Flask為基底，Flask簡單來說就是使用Python寫的輕量級Web應用框架，雖然我也看過有的開發者以Django框架為基底，但是因為Django應用起來會稍微複雜一點，所以我們就像官方一樣使用Flask為基底吧！

首先必須先安裝官方釋出的套件，這樣才能使用相關的功能，當然Flask的套件也必須一併安裝。

安裝好了後我們來看看官方所提供的Usage：

從上往下看，在第15、16行可以看到LineBotApi和WebhookHandler的設置，這個是必須要填入的相關資訊，等於說他要這些資訊（CHANNEL_ACCESS_TOKEN和CHANNEL_SECRET），LineBot才能正確的辨別與發送訊息，這個部分需要注意一下，總之先把整個官方給的Usage複製到一個空白的Python檔並用Visual Studio Code編輯。

CHANNEL_ACCESS_TOKEN和CHANNEL_SECRET的設定

將官方釋出的Usage內容複製到空白的Python檔後就可以開始編輯的動作了，首先須分別填入CHANNEL_ACCESS_TOKEN和CHANNEL_SECRET的設定值，這兩個值分別可以在Line Developers（Line 開發者服務）裡剛新建立的LineBot中找到，點入剛建立好的LineBot後，從Basic Setting裡找到Channel secret與Messaging API裡的Channel access token，分別按下issue後複製到對應的程式碼位置。


Deta Cloud的介紹

Deta Cloud是一個比較年輕的雲端服務平台，若有接觸到這方面的coder們，應該更常聽到Heroku、Google Cloud Platfotm等，我自己的第一個LineBot Api也是部屬Heroku上，但認識Heroku的人都知道Heroku有一些小毛病，Heroku的免費dyno有睡著和超時罷工的問題，雖然這些都可以透過綁定信用卡卡號解決，不過也因為這樣讓建置起來的步驟麻煩了一點，而今天的主角Deta Cloud打著「The cloud for makers and dreamers」與「Deta is free for ever」的口號，就這樣華麗的登場了，看到這樣的口號可以得知，Deta Cloud是免費而且是為了築夢者而建立的雲端服務平台，這對我這個初出茅廬的菜鳥coder來說，目前無疑是最好的端服務平台了。
雖然Deta Cloud是免費使用的，但目前只支援Python與Node.js，且Deta Cloud較為年輕，穩定度可能不如Heroku等較為老練的雲端服務台，所以目前只推薦用於個人的小專案上，希望經過時間歷練的Deta Cloud可以一天比一天茁壯與耀眼！

設定Webhook

Webhook簡單來說就是一串抱著資料的網址，只要將LineBot連上我們相對應的網址就能接收資料，而Webhook的網址在Deta Cloud的平台（Dash board介面）上就能看到，在專案的右上角就能看到一串網址，將那段網址複製下來到Line Developers（Line開發者服務）中設定。

複製好後，在Messaging API選項中可以找到Webhook的設置，只要將剛剛複製的Webhook網址貼上，並在網址後面加上/callback，按下Verify後，若出現Success就代表設定完成，接下來只要掃描上方的QR code加入機器人為好友，就可以得到一個「鸚鵡」機器人了。

結語、小談

這次文章的想法最一開始是，某次因緣際會下認識了Deta Cloud這個雲端服務平台，搭配上自己之前摸索過的LineBot經驗，將這兩個結合在一起，雖然Deta Cloud做為一個新興雲端服務平台，但對於用來建立屬於個人的小專案依然是非常不錯的，往後若搭配LineBot的一般訊息與Flex Message訊息的傳輸等，將可以做到個人電子化名片、記事通知、天氣、甚至是股市訊息推送，不只有無限的潛力，更能便利自己的生活，下此有機會的話再與大家分享（避免自己忘記踩坑）更多LineBot上的運用，而非只是「鸚鵡」機器人，若文章有錯誤的部分或是更好的方法也請在下方留言讓我知道，並一起分享所學的知識，一起加油！！！
